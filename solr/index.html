Solr info page


Searching a Solr Core:

Format:  http://localhost:8983/solr/<CORE-NAME>/browse


Solr Administration:

Solr Admin UI http://localhost:8983/solr
Solr Luke UI Format:  http://localhost:8983/solr/<CORE-NAME>/admin/luke?show=schema&wt=json

To enable the Solr Luke UI, the following needs to be added to the Solr core's "solrconfig.xml" file (e.g. example/solr/collection1/conf/solrconfig.xml):
<requestHandler name="/admin/luke" class="org.apache.solr.handler.admin.LukeRequestHandler" />
(See https://wiki.apache.org/solr/LukeRequestHandler)

Note: If you are using a Solr core that was created using the "my-configset-solr-731" ConfigSet, then the Solr search UI will include a "schema" link to load the Luke UI.

Q & A:

Q A
What Lucene search fields are available? The search fields are defined in the "managed-schema" file in the ConfigSet, so they will end up in core's config files, e.g.:
D:\ds\DocSearcher\solr\solr-7.3.1\server\solr\core_Sampledocs\conf\managed-schema

You can use the "schema" link available from the Solr search UI to see the Luke configuration. However, it is easier to use the Luke standalone application, e.g.:

Note: If it looks like not all of the fields are being displayed, it could be that the last field displayed has a value value. Use the scrollbar on the right to scroll-down the fields.
What is the default Lucene search field used by "my-configset-solr-731" ConfigSet? The "my-configset-solr-731" ConfigSet uses the "text" field.

The default field of text is defined in the "solrconfig.xml" file in the ConfigSet:
<requestHandler name="/query" class="solr.SearchHandler">
	<lst name="defaults">
	<str name="echoParams">explicit</str>
	<str name="wt">json</str>
	<str name="indent">true</str>
	<str name="df">text</str>
	</lst>
</requestHandler>
Note: This is a catch-all field, comprised of copies of pretty much all of the other fields.
Examine the "managed-schema" file to see which fields are "copied" to the "text" field, e.g.:
<copyField source="title" dest="text"/>
<copyField source="author" dest="text"/>
<copyField source="description" dest="text"/>
<copyField source="keywords" dest="text"/>
<copyField source="content" dest="text"/>
