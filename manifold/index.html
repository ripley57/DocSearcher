Indexing changes to existing documents, indexing new documents, and handling removed documents

Button on "Status and Job Management" page What new content is indexed
Start minimal Indexes new files.
Indexes updates to existing files (but not filename changes - a renamed file will be added as a new file).
Start Indexes renamed files.
Indexes deleted files (i.e. they are removed from the index).

Not commonly used indexing options (according to ManifoldCF In Action book)

Button on individual Output Connection page Description, from Manifold In Action book
Re-index all associated documents Make ManifoldCF re-index all documents that were previously indexed, when the associated jobs are run the next time. You would need to click this button only if you had changed the configuration on the target system - perhaps, for example, changing what metadata fields exist.
Remove all associated records Somewhat similar to "Re-index all associated documents", but would be appropriate if you had completely deleted the index on the target system and wanted to start over. These buttons are helpful mainly in a development situation, and should be used with caution in a production system.
